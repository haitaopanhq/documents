# Claim Verification / 事实核验清单

简历先可信，再有说服力。以下内容在当前材料中存在不同版本或需要项目证据支持，投递前必须核验。

| 表述 | 当前材料的风险 | 建议处理 |
|---|---|---|
| “3 个月从需求到生产上线” | AI Delivery 版本描述为 PoC；与生产上线口径不一致 | 未有上线证据时，只写 “delivered a PoC / 完成 PoC 验证” |
| “成本降低 60%”“根因准确率 85%+”“定位缩短 70%” | 没有可见的测量方法、基线或验收材料 | 删除，或补充测量口径、样本、时间与可出示材料 |
| “主导多个企业 AI Agent 平台” | 需明确项目数、角色和交付阶段 | 改为可核验的 “参与/主导 [项目] 的 [职责]” |
| Xstream App、Xworkmate / AIworkspace、svc.plus | 需要公开链接、Demo 或代码说明以支撑“发布/生产级/运营中” | 未公开则写为“独立原型/持续构建”；对 svc.plus 的演示状态与模拟数据不作业务指标 |
| 英语水平 | 不同版本写 Conversational / Professional | 按真实面试与工作能力统一，宁可保守 |
| 雇主与日期 | ECCOM/华讯、Roche、不同开始结束时间在版本间不一致 | 以劳动合同、LinkedIn 或背调资料为唯一来源 |

## 每项成果的最小证据

- **数字成果：** 计算方法、前后基线、时间范围、谁确认。
- **客户项目：** 公司/项目可否公开、本人角色、阶段（方案/PoC/上线）、可说与不可说的边界。
- **个人项目：** 代码或 Demo 链接、截图、README、实际可运行功能。
- **AI 能力：** 不列“精通”清单；准备一个具体案例，说明模型、上下文、工具、评估与失败边界。
